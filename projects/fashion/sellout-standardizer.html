Sell-Out Standardizer — Appendix

What this is

Normalize wide retailer sell-out reports into one analytics-ready table. The tool flattens stacked headers such as WTD / MTD / YTD over duplicated metrics, keeps product fields as dimensions, and saves repeatable mappings.

Quick start

Upload recurring reports or paste an Excel range, confirm the cadence and metric header rows, map product fields, then preview and export the standardized output.

Privacy

All parsing happens locally in your browser. Imported spreadsheets are not uploaded to a server.

01Input

Load weekly sell-out reports into the tool. Each file's sheets become individual sources you can include or exclude.

  • Insert File — opens the system file picker. Accepts .xlsx, .xls, .csv; multi-select is allowed.
  • Drop zone — shows when no sources are loaded. Drag files onto it to import.
  • Files / Paste tabs — switch between uploading files and pasting an Excel range as a single source.
  • File rows — each row is one sheet from one file. Checkbox toggles whether the sheet is included; Remove deletes the source.
  • Clear — drops every loaded source and resets the preview.

02Header Detection

Tell the tool where the header rows are and which cadences and metrics to expect. These settings drive the Field Mapping panel.

  • Cadence Row — 1-based row number that holds cadence labels (set to 0 if cadences are baked into the metric headers themselves).
  • Metric Header Row — 1-based row number for the column names.
  • Report Period — free-text tag written into every output row (e.g. 2026 W21).
  • Delete Rows — Excel row numbers stripped before parsing (e.g. 1, 3-5). Useful for totals or subtitle rows.
  • Ignore Rows — keeps the sheet shape intact but excludes those rows from the output.
  • Cadence Type — multi-select. Choosing WTD + YTD adds those cadences to Field Mapping.
  • Cadence Metrics — multi-select. Each metric × selected cadence produces a mapping row (e.g. WTD Sales $).
  • Map LY values — doubles every metric mapping row with a Last-Year variant and stamps year_basis on output rows.
  • Calculate ST% — emits a calculated sell_through_pct = Sale U / (Sale U + On Hand U) row whenever both inputs exist for the same cadence + year basis.
  • Auto — scans the first 12 rows of the first included sheet and picks the most likely cadence row and metric header row.

03Field Mapping

Bind canonical fields to specific columns in the source. Dimensions stay wide; metric mappings drive the long-format output.

  • Product Dimensions — pick the column that holds each standard product/retail attribute (brand, style, upc, store_name, etc.). Unmapped fields stay blank in the output.
  • Cadence Metrics — one mapping row per cadence × metric × (TY/LY). Driven by the selections in Header Detection. Empty selections produce no output rows for that combination.
  • Remap — clears all current mappings and re-runs auto-matching against header text (cadence keyword + metric aliases + LY presence).

04Preview

Inspect the result before exporting. The first 500 rows are rendered; the full set is still exported.

  • Output tab — the standardized long-format table that will be exported.
  • Headers tab — debug view: every detected column with its detected cadence, year-basis, and current mapping.
  • Raw tab — the first 80 unmodified rows of the first included sheet, so you can sanity-check what was parsed.
  • CSV / JSON / XLSX — download the full standardized output. Buttons stay disabled until at least one mapped metric produces values.

Standard Output

Each metric column becomes one row with source_file, source_sheet, source_row, report_period, cadence, year_basis, metric, and value.

Stacked Headers

When a row above the metric headers contains timing labels such as WTD, MTD, QTD, or YTD, the app carries that cadence across repeated columns like Sale U and Sale $.

LY and ST%

The LY toggle treats LY and Last Year labels as comparison values instead of cadences. The ST% toggle adds calculated sell-through rows when Sale U and an on-hand unit column are available for the same cadence and year basis.

Sheets and Rows

Use sheet checkboxes to include workbook tabs. Delete Rows removes 1-based Excel row numbers before parsing. Ignore Rows keeps the sheet structure intact but skips those original row numbers in the final output.

Configurations

Exporting a config saves detected rows, report period, default cadence, and field mapping so the same retailer report shape can be normalized again.
