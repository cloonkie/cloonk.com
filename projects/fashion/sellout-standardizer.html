Retail Data Standardizer — Appendix

What this is

Normalize wide retailer sell-out reports into one analytics-ready table. The tool flattens stacked headers such as WTD / MTD / YTD over duplicated metrics, keeps product fields as dimensions, and saves repeatable mappings.

Quick start

Upload recurring reports or paste an Excel range, confirm the cadence and metric header rows, map product fields, then preview and export the standardized output.

Privacy

All parsing happens locally in your browser. Imported spreadsheets are not uploaded to a server.

01Input

Load weekly sell-out reports into the tool. Each file's sheets become individual sources you can include or exclude.

  • Insert File — opens the system file picker. Accepts .xlsx, .xls, .csv; multi-select is allowed.
  • Drop zone — shows when no sources are loaded. Drag files onto it to import.
  • Files / Paste tabs — switch between uploading files and pasting an Excel range as a single source.
  • File rows — each row is one sheet from one file. Checkbox toggles whether the sheet is included; Remove deletes the source.
  • Clear — drops every loaded source and resets the preview.

02Header Detection

Tell the tool where the header rows are and which cadences and metrics to expect. These settings drive the Field Mapping panel.

  • Header Group Row - optional 1-based row number that holds cadence labels like WTD, MTD, or YTD.
  • Column Header Row - 1-based row number for the actual field and metric names.
  • Delete Rows - Excel row numbers stripped before parsing, useful for totals or subtitle rows.
  • Cadence Type - multi-select. Choosing WTD + YTD adds those cadence slots to Field Mapping.
  • Cadence Metrics - multi-select. Each metric times cadence creates a mapping row.
  • LY - select it in Cadence Type to add a Last-Year variant for every selected cadence and metric.
  • Calculated Metrics - adds formula-based metrics such as AUR, ST%, or a custom formula using mapped metric and dimension keys.
  • Auto - scans the first 12 rows of the first included sheet and chooses likely header rows.

03Field Mapping

Bind canonical fields to source columns. Dimension groups stay wide; cadence metrics become the long-format fact rows.

  • Retail Context - maps account, channel, location type, and store/door labels.
  • Product Identity - maps brand, style, SKU, UPC, and description fields.
  • Product Attributes - maps color, size, category, season, status, and related product descriptors.
  • Custom dimensions - add extra source attributes that should stay wide on every output row.
  • Cadence Metrics - one mapping row per cadence, metric, and TY/LY basis. Empty selections produce no output rows for that combination.
  • Custom metrics - add extra metric names when a retailer report has fields outside the default metric set.
  • Remap - clears all current mappings and re-runs auto-matching against header text.

04Preview

Inspect the result before exporting. The first 500 rows are rendered; the full set is still exported.

  • Output tab — the standardized long-format table that will be exported.
  • Headers tab — debug view: every detected column with its detected cadence, year-basis, and current mapping.
  • Raw tab — the first 80 unmodified rows of the first included sheet, so you can sanity-check what was parsed.
  • CSV / JSON / XLSX — download the full standardized output. Buttons stay disabled until at least one mapped metric produces values.

Standard Output

The export uses a long metric schema. Product and retail columns stay as dimensions, while each mapped cadence metric becomes one output row.

Schema areaFields
Lineagesource_file, source_sheet, source_row
Retail contextretailer, channel, location_type, store_name
Product identitybrand, style, sku, upc, description
Product attributescolor, size, product_type, material, release, seasonality, sale_status, plus custom dimensions
Metric factcadence, year_basis, metric, value, value_source

Metric rows include sale_units, sale_dollars, on_hand_units, custom metric keys, and optional calculated sell_through_pct.

Stacked Headers

When a row above the metric headers contains timing labels such as WTD, MTD, QTD, or YTD, the app carries that cadence across repeated columns like Sale U and Sale $.

LY and Calculated Metrics

The LY toggle treats LY and Last Year labels as comparison values instead of cadences. Calculated metrics add formula-based rows when the needed mapped fields are available.

Sheets and Rows

Use sheet checkboxes to include workbook tabs. Delete Rows removes 1-based Excel row numbers before parsing.

Configurations

Exporting a config saves detected rows, cadence choices, and field mapping so the same retailer report shape can be normalized again.
